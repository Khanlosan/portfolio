## Опыт

### Mobile QA — медицинский SaaS (Android-приложение)
*1 год*

Тестировал нативное Android-приложение для медицинской CRM (Flutter-based). Сочетал ручной exploratory testing с автоматизацией через adb и Appium.

**Что делал:**
- Прогон высокоприоритетных сценариев по чек-листу: создание/удаление пациентов, счетов, скидок, отчётов, проверка push-уведомлений и сетевых сценариев
- Анализ APK-манифеста (`dumpsys package`, `pm dump`) для проверки наличия фич на уровне ниже UI
- Подготовка баг-репортов с воспроизводимыми шагами, скриншотами и видео-демо (`adb shell screenrecord`)
- Документация технических требований к разработке для перехода к стабильной автоматизации

**Ключевые находки:**

1. **Критический баг ввода в форме платежа.** Поле суммы принимало только цифру `0` и точку — цифры 1-9 игнорировались. Воспроизводилось 100%, блокировало основной флоу. Зафиксировано видео-демо.

2. **Отсутствие функционала в production APK.** Через анализ манифеста обнаружил, что push-уведомления заявлены в чек-листе, но не реализованы в сборке: нет permission `POST_NOTIFICATIONS`, нет FCM-сервисов, нет UI настроек. Escalated в продуктовую команду.

3. **Серия UX-багов в диалогах подтверждения.** Диалоги удаления без контекста (не показывают номер счёта/имя пациента), кнопки `Confirm | Cancel | Delete` визуально равноправны — пользователь может перепутать. Зафиксировано с предложением улучшений.

4. **Тихий cap в поле скидки на 99% без визуальной подсказки.** Пользователь не понимает, почему ввод останавливается.

**Технический gap, который задокументировал для разработки:**
- Flutter-based UI блокирует стандартный adb input на кастомных TextField — `input text`, `keyevent` и тапы по клавиатуре игнорируются
- Для автоматизации требуется: пересборка APK с Flutter Driver Extension (отдельный test-flavor), testID/Semantics на ключевые поля, deeplinks для прыжка в нужное состояние, тестовый backend с сидером данных

## Технические навыки

**Mobile testing:**
- Android (эмулятор Pixel 5 / API 34, физические устройства)
- adb (devices, shell, dumpsys, pm dump, install, push/pull, screenrecord, input tap/swipe/text/keyevent)
- Android Studio + AVD Manager (создание/настройка эмуляторов, изменение Network Speed)
- APK-анализ (permissions, services, receivers, manifest)

**Автоматизация:**
- Appium 2 + WebdriverIO + TypeScript
- UiAutomator2 driver (с учётом ограничений на Flutter)
- Понимание Flutter Driver / `enableFlutterDriverExtension()` как пути к Flutter-автоматизации

**Языки и инструменты:**
- TypeScript / JavaScript (базовый уровень для написания тест-кейсов)
- Python (базовый — скрипты для парсинга и обработки данных, готов углубляться в pytest)
- SQL (базовый — SELECT, JOIN, WHERE, GROUP BY для проверки данных в базе)
- Git, Markdown, HTML (для оформления отчётов)
- PowerShell / Bash (для скриптовых задач)

**Методология:**
- Ручное exploratory testing
- Подготовка чек-листов по приоритетам (H / M / L)
- Воспроизводимые баг-репорты (шаги, ожидание, факт, окружение, доказательства)
- Видео-демо находок (короткие записи через `adb screenrecord`)
- Разделение Pass / Partial / Fail / Not tested без размытых статусов

**Soft skills:**
- Чёткая коммуникация с продуктовой командой и разработкой
- Готовность задавать уточняющие вопросы вместо угадывания требований
- Понимание разницы между «функционал не работает» и «функционал отсутствует»
