# BUG-002 — Push-уведомления отсутствуют в production-сборке

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Severity** | High |
| **Priority** | P2 |
| **Компонент** | Уведомления / Сборка APK |
| **Статус** | Escalated в продуктовую команду |
| **Воспроизводимость** | 100% |
| **Тип** | Функционал отсутствует (не «не работает») |

## Окружение

- Сборка: `app-release.apk` (production-канал), версия `1.x.x`
- Инструменты анализа: `adb shell dumpsys package <package>`, `adb shell pm dump <package>`

## Контекст

Push-уведомления заявлены в чек-листе фич и в требованиях, но при ручной проверке не приходили. Чтобы отличить «баг доставки» от «фичи нет в сборке», проведён анализ манифеста на уровне ниже UI.

## Шаги воспроизведения

1. Установить production-сборку: `adb install app-release.apk`.
2. Выгрузить данные пакета:
   ```
   adb shell dumpsys package <package> > pkg.txt
   ```
3. Проверить наличие разрешения на уведомления, FCM-сервисов и receiver'ов:
   ```
   findstr /i "POST_NOTIFICATIONS" pkg.txt
   findstr /i "firebase messaging FirebaseMessagingService" pkg.txt
   ```
4. В приложении: открыть **Настройки** → найти раздел управления уведомлениями.

## Ожидаемый результат

- В манифесте присутствует permission `android.permission.POST_NOTIFICATIONS` (требуется с Android 13+).
- Зарегистрирован FCM-сервис (наследник `FirebaseMessagingService`).
- В UI есть раздел настроек уведомлений.

## Фактический результат

- Permission `POST_NOTIFICATIONS` **отсутствует** в манифесте.
- FCM-сервисы и receiver'ы для push **не зарегистрированы**.
- В UI **нет** раздела настроек уведомлений.

**Вывод:** функционал push-уведомлений не реализован в production-сборке — это не дефект доставки, а отсутствие фичи. Чек-лист не соответствует фактической сборке.

## Доказательства

- Релевантные выдержки из `dumpsys package` (обезличены) — приложены в трекере.

## Примечания

- Ценность находки: сэкономлено время команды — задачу «починить доставку push» переклассифицировали в «реализовать фичу».
- Рекомендация: синхронизировать чек-лист фич с фактическим scope сборки перед каждым релизом.
