# BUG-003 — Диалог удаления без контекста, кнопки визуально равноправны

| Поле | Значение |
|------|----------|
| **ID** | BUG-003 |
| **Severity** | Medium |
| **Priority** | P3 |
| **Компонент** | UX / Диалоги подтверждения |
| **Статус** | Reported (с предложением улучшений) |
| **Воспроизводимость** | 100% |

## Окружение

- Эмулятор Pixel 5, Android 14 (API 34)
- Сборка: `app-release.apk`, версия `1.x.x`

## Предусловия

В системе есть минимум 2 счёта у разных пациентов.

## Шаги воспроизведения

1. Открыть список счетов пациента.
2. Нажать на иконку удаления у любого счёта.
3. Изучить появившийся диалог подтверждения.

## Ожидаемый результат

- Диалог содержит **контекст**: на удаление какого объекта он ссылается (номер счёта, имя пациента, сумма).
- Деструктивное действие визуально выделено и/или отделено от безопасного (например, `Delete` — красная, `Cancel` — нейтральная).

## Фактический результат

- Текст диалога обезличен: «Вы уверены, что хотите удалить?» — **без указания номера счёта/имени пациента**. Пользователь не видит, что именно удаляет.
- Кнопки **`Confirm | Cancel | Delete`** оформлены одинаково (равный размер, цвет, вес). Высокий риск ошибочного нажатия и необратимого удаления не того объекта.

## Доказательства

- Скриншот диалога (обезличен) — приложен в трекере.

## Предложение по улучшению

1. Добавить в текст диалога идентификатор объекта: `Удалить счёт №{invoice_id} пациента {patient_name} на сумму {amount}?`
2. Оставить две кнопки вместо трёх (`Confirm` и `Delete` дублируют смысл).
3. Деструктивную кнопку — акцентным/красным цветом, безопасную (`Cancel`) — нейтральной и по умолчанию в фокусе.

## Примечания

- Severity Medium: данные не теряются «сами по себе», но дизайн провоцирует ошибку пользователя с необратимыми последствиями (удаление финансового документа).
