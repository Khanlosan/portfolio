# BUG-004 — Тихий cap скидки на 99% без визуальной подсказки

| Поле | Значение |
|------|----------|
| **ID** | BUG-004 |
| **Severity** | Low |
| **Priority** | P4 |
| **Компонент** | Скидки / Форма счёта |
| **Статус** | Reported |
| **Воспроизводимость** | 100% |

## Окружение

- Эмулятор Pixel 5, Android 14 (API 34)
- Сборка: `app-release.apk`, версия `1.x.x`

## Предусловия

Открыта форма счёта с полем **«Скидка, %»**.

## Шаги воспроизведения

1. Установить курсор в поле **«Скидка, %»**.
2. Ввести значение `100`.
3. Наблюдать за поведением поля.

## Ожидаемый результат

При попытке ввести значение выше допустимого пользователь получает **обратную связь**: подсказка/тост вида «Максимальная скидка — 99%» либо подсветка поля с пояснением.

## Фактический результат

- Поле молча обрезает ввод до `99` — ввод третьей цифры просто «не проходит».
- **Никакой подсказки нет.** Пользователь не понимает, почему значение не вводится, и может счесть это зависанием/багом ввода.

## Доказательства

- Короткое видео ввода (обезличено) — приложено в трекере.

## Предложение по улучшению

- Показывать неблокирующую подсказку при достижении лимита: «Максимум 99%».
- Либо разрешить ввод `100` с понятной валидацией при сохранении.

## Примечания

- Severity Low: бизнес-логика (cap на 99%) корректна, проблема исключительно в отсутствии обратной связи. Влияет на доверие к интерфейсу, не на данные.
