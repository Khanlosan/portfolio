# BUG-001 — Поле суммы платежа принимает только `0` и точку

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Severity** | Critical |
| **Priority** | P1 (блокер основного флоу) |
| **Компонент** | Платежи / Форма создания счёта |
| **Статус** | Confirmed |
| **Воспроизводимость** | 100% (10/10) |

## Окружение

- Устройство: эмулятор Pixel 5, Android 14 (API 34); подтверждено на физическом Samsung S7 (Android 8)
- Сборка приложения: `app-release.apk`, версия `1.x.x` (production-канал)
- UI: Flutter-based

## Предусловия

1. Пользователь авторизован под ролью с правом создавать счета.
2. Открыта карточка существующего пациента.

## Шаги воспроизведения

1. Перейти в раздел **Счета → Создать счёт**.
2. Установить курсор в поле **«Сумма»**.
3. Последовательно ввести с клавиатуры цифры: `1`, `2`, `3`.
4. Дополнительно попробовать ввести `5`, `0`, `.`, `7`.

## Ожидаемый результат

Поле принимает любые валидные числовые символы — например, ввод `123` отображает `123`, ввод `50.7` отображает `50.7`.

## Фактический результат

Поле **игнорирует цифры 1–9**. Принимаются и отображаются только символ `0` и точка `.`.
Результат ввода из шага 3 — поле пустое. Результат ввода `5 0 . 7` — в поле остаётся `0.`.
Создать счёт с реальной суммой **невозможно** — флоу оплаты заблокирован.

## Доказательства

- Видео-демо: `adb shell screenrecord` (10 сек, полный цикл ввода) — *приложено в трекере, не публикуется (NDA)*.
- Лог ввода через adb для воспроизведения:
  ```
  adb shell input tap <x> <y>      # фокус на поле "Сумма"
  adb shell input text "123"        # символы игнорируются на уровне Flutter TextField
  ```

## Примечания

- Стандартный `adb input text` на этом Flutter TextField не срабатывает (см. [technical-docs/automation-readiness-flutter.md](../technical-docs/automation-readiness-flutter.md)), поэтому баг проверялся также ручным вводом с экранной клавиатуры — поведение идентично.
- Предположительная причина (для разработки): некорректный `inputFormatter` / неверная маска или regex-фильтр на поле, отсекающий все цифры кроме `0`.
- Влияние: блокирует ключевой бизнес-сценарий (выставление счёта пациенту).
